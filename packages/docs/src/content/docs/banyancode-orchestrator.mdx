# BanyanCode Orchestrator

The orchestrator is a primary agent that decomposes complex tasks, fans out to specialized subagents in parallel, and coordinates them via peer messaging and shared memory.

## Overview

The orchestrator activates when you use the `tab` key to cycle to it, or when you mention `@orchestrator` in your prompt. It analyzes the current task and decides whether to handle it directly or fan out to subagents.

**Fan-out limits:**
- Default: 3 concurrent subagents
- Maximum: 5 subagents (configurable)

When a task exceeds the fan-out limit, the orchestrator escalates to the user for guidance.

## Usage

### Activating the orchestrator

Cycle to the orchestrator using `tab` in the TUI, or mention `@orchestrator` in your prompt:

```
@orchestrator Implement the user auth flow and the session management
```

### When the orchestrator fans out

The orchestrator fans out when a task has independent subtasks that can run in parallel:

- Explore multiple directories or files simultaneously
- Implement multiple features that do not depend on each other
- Research different topics in parallel

### When the orchestrator escalates

The orchestrator escalates to the user when:
- The task exceeds the fan-out limit (more than 5 subagents needed)
- A subagent task fails and requires user intervention
- The task is ambiguous and needs clarification

## Tools available to orchestrator

The orchestrator has access to shared tools for coordinating subagents:

| Tool | Description |
|------|-------------|
| `shared_memory` | Ephemeral state sharing between subagents |
| `subagent_message` | Peer-to-peer messages between subagents |
| `memory_store` | Store data in cross-session memory |
| `memory_recall` | Retrieve data from cross-session memory |
| `memory_list` | List keys in a memory scope |
| `memory_forget` | Remove a key from memory |
| `memory_search` | Search memory with optional embeddings |
| `codegraph_build` | Build a polyglot code graph |
| `codegraph_query` | Query the code graph |
| `codegraph_callers` | Find callers of a function |
| `codegraph_dependents` | Find dependents of a node |
| `codegraph_impact` | Analyze impact of a change |
| `system_status` | Get system health information |

The `/codegraph-build` command runs the `codegraph_build` tool directly. It does not prompt the agent — the build happens in the background, and a progress widget in the TUI shows the status. See [banyancode-config](./banyancode-config.mdx#background-codegraph-build).

## Choosing a model

The orchestrator and each subagent can use a different model. The TUI exposes this via the `/agent-model` slash command.

### Picking a model from the TUI

1. Press `<leader>:` (or type `/`) to open the command palette
2. Type `agent-model` and press Enter
3. Select a model from the list
4. The choice persists to `banyancode.json` and applies to the next orchestrator run

### Manual configuration

You can also set per-agent models directly in `banyancode.json`:

```json
{
  "agent": {
    "orchestrator": { "model": "anthropic/claude-3-5-sonnet" },
    "researcher": { "model": "openai/gpt-4o-mini" },
    "explore": { "model": "anthropic/claude-3-5-haiku" },
    "general": { "model": "anthropic/claude-3-5-sonnet" }
  }
}
```

### When to assign different models

- **Orchestrator:** Use a strong reasoning model (e.g., `claude-3-5-sonnet`) since it plans and synthesizes
- **Researcher:** Use a fast, cheap model with good web search synthesis (e.g., `gpt-4o-mini`)
- **Explore:** Use the cheapest model that can read code (e.g., `claude-3-5-haiku`)
- **General:** Use the same as the orchestrator or a strong all-rounder

## Troubleshooting

### Orchestrator not appearing

Ensure `BANYANCODE_ENABLE=1` is set in your environment. Without this flag, the orchestrator is not registered as an agent.

### Subagent tasks fail

- Verify that `shared_memory` is working correctly — subagents rely on it for coordination
- Ensure fire-and-persist messaging is functioning — messages are durable but not acknowledged
- Check that the subagent mesh is healthy using `Banyan.SubagentBus.peers()`

### Too many subagents

When a task requires more than 5 subagents, the orchestrator escalates to the user. Consider breaking the task into smaller pieces or handling some subtasks directly.
