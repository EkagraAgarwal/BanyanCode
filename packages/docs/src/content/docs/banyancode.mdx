# BanyanCode

BanyanCode is a CLI/TUI-only fork of [OpenCode](https://github.com/anomalyco/opencode) that adds a parallel subagent mesh, cross-session memory, a code graph utility, and a researcher agent with free web search. It is designed for developers who want orchestrated multi-agent workflows with persistent memory across sessions.

## What BanyanCode adds over OpenCode

- **Orchestrator + subagent mesh.** A primary `orchestrator` agent decomposes complex tasks, fans out to specialized subagents in parallel, and coordinates them via peer messaging and shared memory.
- **Cross-session memory.** A persistent key-value store with JSONB indexable payloads, exposed as tools and a skill.
- **Codebase utility.** Builds a polyglot code graph using tree-sitter. `/codegraph-build` or `opencode codegraph build` runs in the background; a floating TUI widget shows progress, current file, and lets you cancel. See [banyancode-codegraph](./banyancode-codegraph.mdx) and [banyancode-config](./banyancode-config.mdx#background-codegraph-build).
- **Researcher agent with free web search.** A `researcher` subagent that uses DuckDuckGo-backed `websearch_free` by default.

## Quick start

### From source

```bash
bun install
bun dev
```

BanyanCode features are on by default. Set `BANYANCODE_ENABLE=0` to run as upstream OpenCode.

### Standalone binary

```bash
cd packages/opencode
bun run script/build.ts -- --single
install -m 0755 dist/opencode-<platform>-<arch>/bin/banyancode ~/.local/bin/
# or on Windows: copy dist\opencode-windows-x64\bin\banyancode.exe to a PATH directory
```

Then run `banyancode` in any project directory. The binary is fully self-contained — no `node_modules` or Bun runtime required.

## When to use the orchestrator vs build/plan

Use the **orchestrator** when:
- A task decomposes into independent parallel subtasks (e.g., explore multiple files, implement multiple features)
- You want the system to coordinate subagents automatically
- You need fan-out from a single prompt

Use **build/plan** directly when:
- You want single-agent focused work
- The task is linear and does not benefit from parallel fan-out
- You prefer direct control over agent behavior

The orchestrator activates via `tab` cycle or `@orchestrator` mention in the prompt.
