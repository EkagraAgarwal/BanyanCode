# BanyanCode

BanyanCode is a CLI/TUI-only fork of [OpenCode](https://github.com/anomalyco/opencode) that adds a parallel subagent mesh, cross-session memory, a code graph + embeddings utility, and a researcher agent with free web search. It is designed for developers who want orchestrated multi-agent workflows with persistent memory across sessions.

## What BanyanCode adds over OpenCode

- **Orchestrator + subagent mesh.** A primary `orchestrator` agent decomposes complex tasks, fans out to specialized subagents in parallel, and coordinates them via peer messaging and shared memory.
- **Cross-session memory.** A persistent key-value store with optional embeddings for semantic search, exposed as tools and a skill.
- **2-phase codebase utility.** Phase 1 builds a polyglot code graph using tree-sitter. Phase 2 computes embeddings over the same nodes for semantic search. `/codegraph-build` runs in the background; a floating TUI widget shows progress, current file, and lets you cancel. See [banyancode-config](./banyancode-config.mdx#background-codegraph-build).
- **Researcher agent with free web search.** A `researcher` subagent that uses DuckDuckGo-backed `websearch_free` by default.

## Quick start

```bash
bun install
bun dev
```

BanyanCode features are opt-in. Set `BANYANCODE_ENABLE=1` in your environment to enable them.

## When to use the orchestrator vs build/plan

Use the **orchestrator** when:
- A task decomposes into independent parallel subtasks (e.g., explore multiple files, implement multiple features)
- You want the system to coordinate subagents automatically
- You need fan-out from a single prompt

Use **build/plan** directly when:
- You want single-agent focused work
- The task is linear and does not benefit from parallel fan-out
- You prefer direct control over agent behavior

The orchestrator activates via `tab` cycle or `@orchestrator` mention in the prompt.
